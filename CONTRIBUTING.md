# Contributing

For general Falco contribution guidelines, including DCO sign-off, see the organization-wide [CONTRIBUTING.md](https://github.com/falcosecurity/.github/blob/main/CONTRIBUTING.md).

## Chart Contributions

Some charts are maintained directly in this repository. Others are synchronized from their source repositories by Falco infrastructure. The managed chart list is defined in [`.github/managed-charts.yaml`](.github/managed-charts.yaml).

For charts maintained directly in this repository:

- update the chart under [`charts/`](charts/);
- bump `Chart.yaml` `version`;
- update `CHANGELOG.md`;
- update `README.gotmpl` and run `make docs` when values or chart docs change.

For charts listed in [`.github/managed-charts.yaml`](.github/managed-charts.yaml), author chart changes in the source repository instead. That includes chart templates, values, `Chart.yaml`, `CHANGELOG.md`, `README.gotmpl`, and generated chart docs. Poiana will sync the resulting chart release PR back to this repository.

Direct non-automation PRs changing managed chart directories are blocked by CI. Automation PRs generated by Poiana are allowed.

Open managed chart bugs, feature requests, and chart-change PRs in the source repository:

| Chart | Source repository |
|-------|-------------------|
| [`falco`](charts/falco) | [`falcosecurity/falco`](https://github.com/falcosecurity/falco/tree/master/chart/falco) |
| [`falco-operator`](charts/falco-operator) | [`falcosecurity/falco-operator`](https://github.com/falcosecurity/falco-operator/tree/main/chart/falco-operator) |
| [`event-generator`](charts/event-generator) | [`falcosecurity/event-generator`](https://github.com/falcosecurity/event-generator/tree/main/chart/event-generator) |
| [`k8s-metacollector`](charts/k8s-metacollector) | [`falcosecurity/k8s-metacollector`](https://github.com/falcosecurity/k8s-metacollector/tree/main/chart/k8s-metacollector) |

Use this repository for issues about the published chart repository, chart index, packaged releases, sync PRs, or charts maintained directly here.
